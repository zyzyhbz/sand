# 需求规格说明书 - 沙箱安全分析系统 PDF报告下载功能

## 1. 项目概述

### 1.1 背景

当前沙箱安全分析系统（Sand Security Analysis System）生成的报告文件为HTML格式，用户在查看报告后可以下载为HTML文件。为了提供更好的阅读体验和跨平台兼容性，需要增加PDF格式的下载选项。

### 1.2 目标

在现有报告查看页面增加PDF格式下载选项，使用户可以下载与HTML内容一致的PDF格式报告文件。

### 1.3 范围

**包含的功能：**
- 在报告查看页面增加PDF下载按钮
- 服务端HTML到PDF的转换功能
- PDF文件直接下载功能

**不包含的功能：**
- 前端PDF预览功能
- PDF批量下载功能
- 邮件发送PDF报告功能

## 2. 功能需求

### 2.1 用户角色

| 角色名称 | 描述 | 权限 |
|----------|------|------|
| 普通用户 | 使用系统进行邮件/附件安全分析的用户 | 查看报告、下载HTML/PDF报告 |

### 2.2 功能清单

#### 2.2.1 PDF下载按钮

- **需求ID**: FR-001
- **需求描述**: 在现有报告查看页面增加PDF格式下载按钮，按钮应与现有的HTML下载按钮并列显示
- **优先级**: 高
- **验收标准**: 
  - 报告查看页面包含"下载PDF"按钮
  - 按钮点击后触发PDF下载流程
  - 按钮应有明确的图标和文字标识
- **依赖关系**: FR-002

#### 2.2.2 服务端PDF转换功能

- **需求ID**: FR-002
- **需求描述**: 实现服务端HTML到PDF的转换功能，将现有HTML报告内容转换为PDF格式
- **优先级**: 高
- **验收标准**:
  - 能够将HTML报告内容完整转换为PDF
  - PDF内容与HTML报告内容一致
  - PDF文件可读、格式清晰
- **依赖关系**: 无

#### 2.2.3 转换失败处理

- **需求ID**: FR-003
- **需求描述**: 处理PDF转换过程中可能出现的失败情况
- **优先级**: 中
- **验收标准**:
  - 转换失败时返回友好的错误提示
  - 错误信息记录日志
  - 不影响用户继续使用HTML下载功能

#### 2.2.4 PDF下载API

- **需求ID**: FR-004
- **需求描述**: 新增PDF格式下载的API端点
- **优先级**: 高
- **验收标准**:
  - API端点格式：`GET /api/report/:id/download/pdf`
  - 返回Content-Type为application/pdf
  - 文件名包含.pdf后缀
- **依赖关系**: FR-002

## 3. 用户故事

### 3.1 下载PDF报告

**作为** 普通用户  
**我想要** 在查看报告后能够下载PDF格式的报告  
**以便于** 在没有网络的环境下阅读或存档报告

**验收条件**:

* 在报告查看页面可以看到"下载PDF"按钮
* 点击按钮后浏览器开始下载PDF文件
* 下载的PDF文件内容与HTML报告一致
* PDF文件可以在常用PDF阅读器中正常打开

### 3.2 转换失败时获得提示

**作为** 普通用户  
**我想要** 在PDF转换失败时获得明确的错误提示  
**以便于** 了解问题原因并选择其他方式获取报告

**验收条件**:

* 转换失败时页面显示错误提示信息
* 错误提示说明失败原因
* 用户仍可选择下载HTML格式报告

## 4. 数据需求

### 4.1 数据实体

- **报告JSON数据**: 包含分析结果的JSON数据结构，已存在于现有系统中
- **HTML报告内容**: 由现有generateHTMLReport函数生成的HTML字符串
- **PDF文件**: 转换后的PDF二进制数据

### 4.2 数据流

1. 前端发起PDF下载请求 → 服务端接收请求
2. 服务端读取报告JSON数据 → 生成HTML内容
3. HTML内容转换为PDF格式 → 返回给前端下载

## 5. 假设和依赖

### 5.1 假设

- HTML报告生成功能正常工作
- 服务端有足够的资源进行HTML到PDF的转换
- 转换库能够正确处理HTML中的CSS样式

### 5.2 依赖

- 报告数据存储在 `/reports` 目录的JSON文件中
- 现有HTML报告生成逻辑位于 `routes/report.js` 的 `generateHTMLReport` 函数
- PDF转换库的选择需要考虑Windows环境兼容性

## 6. 技术约束

### 6.1 转换方案约束

- 推荐使用服务端转换方案
- 需考虑Windows环境下的兼容性
- 转换库需要支持HTML到PDF的完整转换

### 6.2 性能约束

- PDF转换应在合理时间内完成（建议不超过30秒）
- 大型报告可能需要更长的处理时间，应有适当的超时设置
