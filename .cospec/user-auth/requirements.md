# 用户登录与数据隔离系统 - 需求文档

## 1. 概述

### 1.1 项目背景
当前邮件安全检测系统无用户认证机制，所有用户共享同一套数据（上传文件、分析报告等）。需要引入用户登录系统，实现多用户独立使用，各自拥有独立的后台数据。

### 1.2 目标
- 提供用户注册和登录功能
- 实现用户间数据完全隔离（文件、报告、分析记录）
- 保持现有功能的完整性和一致性

## 2. 功能需求

### 2.1 用户注册
- **FR-001**: 用户可通过用户名、密码进行自助注册
- **FR-002**: 用户名唯一性校验，不可重复
- **FR-003**: 密码强度要求：至少6位字符
- **FR-004**: 注册成功后自动登录并跳转到主页
- **FR-005**: 注册失败时给出明确错误提示（用户名已存在、密码过短等）

### 2.2 用户登录
- **FR-006**: 已注册用户通过用户名、密码登录
- **FR-007**: 登录成功后跳转到主页面
- **FR-008**: 登录失败时给出错误提示（用户名或密码错误）
- **FR-009**: 支持"记住我"功能（延长Token有效期）

### 2.3 会话管理
- **FR-010**: 基于 JWT Token 的无状态认证
- **FR-011**: Token 有效期：普通登录2小时，"记住我"7天
- **FR-012**: Token 过期后自动跳转到登录页
- **FR-013**: 用户可主动退出登录

### 2.4 用户信息展示
- **FR-014**: 登录后在导航栏显示当前用户名
- **FR-015**: 提供用户菜单（退出登录）

### 2.5 数据隔离
- **FR-016**: 每个用户拥有独立的上传文件目录
- **FR-017**: 每个用户只能查看和管理自己的报告
- **FR-018**: 用户间文件和报告数据完全隔离
- **FR-019**: 未登录用户无法访问主页面和API接口（登录/注册接口除外）

### 2.6 认证中间件
- **FR-020**: 所有受保护的API路由需验证JWT Token
- **FR-021**: Token验证失败返回401状态码
- **FR-022**: 前端接收到401自动跳转登录页

## 3. 非功能需求

### 3.1 安全性
- **NFR-001**: 密码使用 bcrypt 加盐哈希存储，不可明文
- **NFR-002**: JWT Secret 从环境变量读取
- **NFR-003**: 敏感接口防暴力破解（登录频率限制）

### 3.2 可用性
- **NFR-004**: 登录/注册页面UI风格与现有页面一致（浅蓝色矩形设计）
- **NFR-005**: 操作响应流畅，无明显延迟

## 4. 约束

- **CON-001**: 数据库使用 SQLite（轻量级，无需外部数据库服务）
- **CON-002**: 认证方案使用 JWT（无状态，适合当前Express架构）
- **CON-003**: 独立登录页面（`login.html`），非弹窗形式
- **CON-004**: 沿用现有前端技术栈（Bootstrap 5 + 原生JavaScript）
- **CON-005**: 新增npm依赖应最小化

## 5. 影响范围

### 5.1 新增文件
- `public/login.html` - 登录/注册页面
- `routes/auth.js` - 认证路由（登录、注册、退出）
- `services/userService.js` - 用户服务（CRUD、密码验证）
- `middleware/auth.js` - JWT认证中间件
- `database/` - SQLite数据库目录

### 5.2 需修改文件
- `server.js` - 新增认证路由、数据库初始化、路由保护
- `package.json` - 新增依赖（better-sqlite3, bcryptjs, jsonwebtoken）
- `.env` - 新增JWT_SECRET配置
- `routes/report.js` - 报告数据关联userId
- `routes/upload.js` - 上传文件关联userId、按用户目录存储
- `routes/sandbox.js` - 分析接口增加认证
- `routes/ai.js` - AI接口增加认证
- `routes/attachment.js` - 附件接口增加认证
- `public/index.html` - 导航栏添加用户信息、前端Token管理
- `public/js/app.js` - 所有API调用增加Token header、401处理

## 6. 验收标准

1. 用户可以注册新账号
2. 用户可以登录系统
3. 登录后能看到自己的用户名
4. 用户A上传的文件对用户B不可见
5. 用户A生成的报告对用户B不可见
6. 未登录访问主页自动跳转登录页
7. Token过期后自动跳转登录页
8. 用户可以正常退出登录
